# 大模型


全称大语言模型（Large Language Model, LLM），是一种人工神经网络构建的，基于深度学习的自然语言处理系统，其核心建立在Transformer架构之上、拥有海量参数的人工智能模型。

## 1.1 核心架构：Transformer 与 自注意力机制

目前LLM几乎都是基于Transformer架构。其最关键的创新在于自注意力机制。

- 数学本质：它允许模型在处理输入序列的每一个 token 时，计算该 token 与序列中所有其他 token 的相关性权重。
- 计算过程：通过查询（Query）、键（Key）和值（Value）的交互来计算上下文表示。公式表达为：



$Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}})V$


其中  $d_k$  是键向量的维度。除以  $ \sqrt{d_k} $  是为了防止点积过大导致 Softmax 函数进入梯度极小的区域，从而影响反向传播的效率。

- 位置编码：由于 Transformer 本身是并行处理且不具备循环神经网络（RNN）那样的时序归纳偏置，因此必须引入位置编码（Positional Encoding）或位置嵌入（Positional Embedding），将 token 在序列中的位置信息注入到输入向量中。


## 1.2 训练目标：自回归语言建模

LLM 的基础训练任务通常是自回归语言建模。

- 目标函数：模型旨在学习一个概率分布，即在给定前 $t-1$ 个 token 的情况下，预测第 $t$个token 的概率。
- 优化目标：最大化训练语料库中序列的对数似然函数。对于一个长度为 
$T$ 的序列$x=(x_1,x_2,...,x_T)$,目标是最大化：$L(\theta)$

$\mathcal{L}(\theta) = \sum_{t=1}^{T} \log p(x_t \mid x_{<t};\theta)$

- 损失函数：在实践中，这通常通过交叉熵损失函数来实现。
- LLM并不是"思考后回答"，而是逐token概率采样生成。所有"智能、推理、逻辑"都是从这个朴素目标中涌现出来的。这就直接决定了：会出现幻觉、会重复、长文本会丢失上下文、temperature控制随机性。


## 1.3 扩展定律

这是理解现代 LLM 能力涌现的关键。研究表明，模型的性能（通常以测试集上的损失来衡量）可以预测为计算量、数据集大小和模型参数量的幂律函数。

- 这意味着，随着模型参数量$（ N ）$、数据量$（ D ）$和计算量$（ C ）$的增加，模型性能会以可预测的方式提升，直到达到饱和。
- 这也解释了为什么现在的模型追求“更大”——因为规模本身会带来能力的质变（例如推理能力、代码生成能力等）。


### 1.4 总结

把 LLM 看作是一个拥有海量参数 $θ$的高维函数$f(x)$ ，它通过 Transformer 架构将离散的文本序列映射到连续的高维向量空间，并通过自回归的方式在这个空间中寻找概率最高的下一个状态。
