
# Prompt

## 了解

大语言模型里面是海量的数据，回答问题具有随机性和不确定性，很大可能无法直接精确匹配实际应用需求。那么这时候就需要Prompt Engineering（提示工程），它可以在不改变模型与训练权重前提下，通过结构化、精准化的文本指令，解决任务表达不清、输出跑偏、输出格式不稳定、模型过度发散、业务目标和模型回答不一致。

> Prompt 工程的核心价值是：降低歧义，提升任务对齐程度。用更清晰的上下文设计去控制一个概率型系统。


Prompt不等于"用户问的那句话"，而是System指令、用户问题、历史对话、示例、检索资料、工具说明、输出格式要求。也就是"Prompt是模型在当前轮次看到的任务上下文"。

好的Prompt由四个核心组成：

- 角色：告诉模型它现在是什么身份
- 任务：这次具体要完成什么
- 约束：告诉模型哪些边界必须遵守
- 输出格式：告诉模型输出结果长什么样

实例：

```text
讲一下 RAG。
```
这个就是弱Prompt很广泛，没有受众、没有深度要求、没有输出结构要求、没有约束。

```text
你是一名面向前端工程师教学的 AI 讲师。
请用中文解释什么是 RAG。
要求：
- 先给一句话定义
- 再说明它解决什么问题
- 最后举一个企业知识问答的例子
- 不要使用太多术语
```

上面这个Prompt更好，也符合工程式Prompt，因为更可控、更清楚、更贴近目标。

:::warning
Prompt不是越长越好，要越清楚越好。<br/>
Prompt不能能够解决所有问题，有时候也需要配合工具。
:::
